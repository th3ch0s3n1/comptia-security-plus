# 🥷 Section 3: Threat Actors & Motivations


> Cornell Notes — CompTIA Security+ (SY0-701) | Maps to Domain 2.0 (Threats, Vulnerabilities, and Mitigations)


## ❓ Cue / Key Questions & Notes


| Question / Keyword | Answer |
| --- | --- |
| **🟢 Threat Actors** |  |
| 🔴 What is a threat actor? | An individual or group responsible for a malicious act against systems or data. |
| 🔴 What are the primary threat actor attributes? | Location (internal vs. external), resources/funding, and level of sophistication/capability. |
| 🔴 What are common threat actor motivations? | Financial gain, espionage, hacktivism, revenge, disruption/chaos, war, or blackmail. |
| 🔴 What is data exfiltration? | The unauthorized transfer of sensitive data out of an organization. |
| **🟢 Threat Actor Types** |  |
| 🔴 What is an unskilled attacker? | Often called a "script kiddie"—an attacker with limited technical skill who relies heavily on pre-made tools or scripts. |
| 🔴 What is a hacktivist? | An attacker motivated by political, social, or ideological causes (e.g., Anonymous). |
| 🔴 What is organized crime in security? | Well-funded, highly sophisticated criminal syndicates driven strictly by financial profit. |
| 🔴 What is a nation-state actor? | A government-sponsored group with massive funding and advanced capabilities, targeting strategic or geopolitical assets. |
| **🟢 Advanced Persistent Threat** | APT |
| 🔴 What is an APT? | A highly sophisticated, prolonged, and stealthy cyberattack designed to maintain long-term, undetected access to a network. |
| **🟢 Internal Risks** |  |
| 🔴 What is an insider threat? | A trusted employee, contractor, or partner who misuses their authorized access to cause harm. |
| 🔴 What is Shadow IT? | Hardware, software, or cloud services deployed within an organization without IT department approval or oversight. |
| **🟢 Attack Infrastructure** |  |
| 🔴 What is a threat vector? | The specific pathway or method an attacker uses to gain entry to a target system (e.g., email, USB, wireless). |
| 🔴 What is an attack surface? | The sum total of all physical and digital points where an unauthorized user can try to enter or extract data. |
| **🟢 Deception Technology** |  |
| 🔴 What is a honeypot? | A single isolated decoy system designed to attract, trap, and analyze malicious activity. |
| 🔴 What is a honeynet? | A dummy network consisting of multiple honeypots that simulates a real, production environment. |
| 🔴 What is a honeyfile? | A decoy file placed on a network that triggers an immediate security alert the moment it is opened or accessed. |
| 🔴 What is a honeytoken? | Fake data records (such as dummy credentials or database entries) used to detect unauthorized access or data exfiltration. |


## 📝 Summary


Defending an infrastructure requires analyzing **threat actors** based on their attributes (internal/external presence, resources, sophistication) and core motivations (profit, espionage, ideology, war). Adversaries range from low-skill **script kiddies** and ideological **hacktivists** to profit-focused **organized crime** and state-sponsored groups deploying sophisticated **Advanced Persistent Threats (APTs)**. While **insider threats** and unmanaged **Shadow IT** compromise the perimeter from within, external actors target specific **threat vectors** to breach the broader **attack surface**. To counter this, security teams deploy **deception technology**—such as **honeypots**, **honeynets**, **honeyfiles**, and **honeytokens**—to safely intercept, isolate, and study malicious activity early.
